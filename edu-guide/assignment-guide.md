# Assignment

## Context
Cloud and/or Edge environment

## Goal
Addressing data management fairness as a sustainability-quality concern

---

## Tasks
1. Imagine one use case scenario that helps you identifying the relevance for fairness and defining how fairness-oriented system features could be defined in the context of the given case.  
2. Create your own definition of fairness and its sub-characteristics, and document it by using Decision Map + SQ model  
3. Sketch such fairness definition as a sustainability perspective  
4. Create(*) a fairness indicator operationalizing your fairness definition, and document it by using KPI canvas  
5. Create(*) one architecture tactic as part of your perspective, and explain how it helps addressing your fairness definition  
6. Create a view illustrating how the tactic would support your own fairness definition / sub-characteristics and sustainability aspects. If useful, you can use elements of the use case scenario  

(*) individual contribution
